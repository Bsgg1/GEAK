# Logging HIP Activity

HIP provides a logging mechanism to trace API and runtime code execution when running HIP applications. This feature helps developers debug issues and provides valuable telemetry to the HIP development team.

## Logging Level

HIP logging is disabled by default. Enable it using the `AMD_LOG_LEVEL` environment variable, which controls the verbosity level:

```
LOG_NONE        = 0
LOG_ERROR       = 1
LOG_WARNING     = 2
LOG_INFO        = 3
LOG_DEBUG       = 4
LOG_EXTRA_DEBUG = 5
```

> "All information under the value set for `AMD_LOG_LEVEL` is printed."

## Logging Mask

The logging mask filters which functionality types to report. After setting `AMD_LOG_LEVEL`, the default mask value is `0x7FFFFFFF`. Customize it via the `AMD_LOG_MASK` environment variable:

| Mask | Value | Purpose |
|------|-------|---------|
| LOG_API | 0x1 | API calls |
| LOG_CMD | 0x2 | Kernel and copy commands |
| LOG_WAIT | 0x4 | Synchronization operations |
| LOG_AQL | 0x8 | AQL packet decoding |
| LOG_QUEUE | 0x10 | Queue commands and contents |
| LOG_MEM | 0x20000 | Memory allocation |
| LOG_TS | 0x80000 | Timestamp details |

## Logging Command

Use the `ClPrint()` macro to output logging information:

```c
ClPrint(amd::LOG_INFO, amd::LOG_INIT, "Initializing HSA stack.");
```

## Practical Examples

**Linux example:**
```bash
export AMD_LOG_LEVEL=4
./hipinfo
```

**Windows example:**
```cmd
set AMD_LOG_LEVEL=4
hipinfo
```

## Logging hipcc Commands

To inspect compiler commands generated by hipcc, set `HIPCC_VERBOSE=1`:

```bash
export HIPCC_VERBOSE=1
hipcc main.cpp
```

This displays the underlying clang++ or NVCC commands to stderr.
